# 🛡️ TASK VERIFIER
## Safe Execution & Regression Protection Protocol

### 🎯 Mission
No task is considered complete without full impact validation.

---

# 🧠 STEP 1 — IMPACT ANALYSIS

Before implementing:

- What files will change?
- Does it affect shared components?
- Does it affect routing?
- Does it affect state?
- Does it affect global styles?

Document impact before coding.

---

# 🔧 STEP 2 — SAFE IMPLEMENTATION

- Modify minimal required files
- Reuse existing structure
- Avoid duplication
- Follow design & architecture guardians

---

# 🔄 STEP 3 — REGRESSION CHECK

Verify:

✔ Header consistency  
✔ Footer consistency  
✔ Navigation styling  
✔ Theme consistency  
✔ Responsive behavior  
✔ No layout overflow  
✔ No performance regression  

---

# ⚡ STEP 4 — PERFORMANCE CHECK

- Avoid unnecessary re-renders
- No large image misuse
- No blocking logic
- No heavy synchronous loops

---

# 🔍 STEP 5 — FINAL VALIDATION

Before closing task:

✔ All guardians passed  
✔ No UI break  
✔ No logic break  
✔ No console errors  
✔ No design conflict  

Only then → Mark as completed.

---

# 🚨 CRITICAL RULE

Never prioritize speed over system stability.

System integrity > Task completion speed.
